## 沟通规范

### Issue管理

```markdown
**问题描述**
IOC启动后立即崩溃，错误信息：Segmentation fault

**复现步骤**
1. 编译IOC：`make`
2. 运行：`./st.cmd`
3. 观察崩溃

**环境信息**
- OS: Ubuntu 20.04
- EPICS Base: 3.15.6
- 架构: x86_64

**期望行为**
IOC正常启动，无错误

**实际行为**
IOC崩溃

**日志**
\`\`\`
./st.cmd
Segmentation fault (core dumped)
\`\`\`
```

### 会议规范

#### 每日站会（15分钟）

- 昨天完成了什么
- 今天计划做什么
- 遇到什么阻碍

#### 迭代回顾

- 做得好的地方
- 需要改进的地方
- 下次行动计划

## 知识分享

### 技术文档

- 定期更新文档
- 记录设计决策
- 分享最佳实践

### Code Review

- 每个PR至少2人审查
- 及时给出反馈
- 相互学习

## 团队规范

### 工作流程

```
计划 → 设计 → 开发 → 测试 → 部署 → 监控
```

### 责任分工

- 开发人员：编写代码和测试
- 审查人员：代码审查
- 测试人员：集成测试
- 运维人员：部署和监控
